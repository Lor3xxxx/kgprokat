# KYRGYZ PROKAT — система аренды строительного оборудования

Реализация по ТЗ (Этапы 1 + 2): публичный каталог с бронью через WhatsApp + рабочее место менеджера
(выдача/возврат, клиенты, рейтинг доверия, дифференцированный залог) + админка правил аренды.

## Стек
- **Next.js 16** (App Router, React 19) + **TypeScript** + **Tailwind CSS 4**
- **Prisma 6** + **SQLite** (файл `prisma/dev.db`)
- Авторизация — JWT в httpOnly-cookie (`jose`), пароли — `bcryptjs`
- QR-коды — `qrcode`

## Запуск
```bash
npm install
npx prisma migrate dev      # создать БД и применить схему
npm run seed                # наполнить демо-данными
npm run dev                 # http://localhost:3000
```

## Каталог
Реальный каталог импортирован с **kyrgyzprokat.kg** (WooCommerce Store API): 274 позиции,
фото скачаны локально в `public/equipment/`, цена за сутки = тариф «Сутки до 24ч».
Цена за неделю и залог — наши значения (правятся в админке).
Повторный импорт: `npx tsx scripts/import-woo.ts` (нужны файлы выгрузки в `.import/`).

## Демо-доступы (рабочее место — /login)
- **Админ:** `admin` / `admin123`
- **Менеджер:** `manager` / `manager123`

## Пользователи и безопасность входа
- Админ создаёт/редактирует/удаляет аккаунты менеджеров в разделе **«Пользователи»** (`/manage/users`).
- **1 аккаунт = 1 устройство:** аккаунт автоматически привязывается к устройству при первом входе (cookie `kp_device`, httpOnly, 5 лет). С другого устройства вход невозможен даже с верным паролем. Админ может **сбросить привязку** (новый компьютер) — аккаунт привяжется заново.
- **Защита от подбора пароля:** после 5 неудачных попыток логин блокируется на 15 минут (в памяти процесса).
- Аккаунт можно отключить (`active`); нельзя удалить/разжаловать последнего админа или отключить самого себя.
- Если устройство админа утеряно — сбросить привязку можно из БД: `UPDATE User SET deviceId = NULL`.

## Безопасность сайта
- **HTTP-заголовки** (`next.config.ts`): Content-Security-Policy, `X-Frame-Options: DENY` (анти-кликджекинг), `X-Content-Type-Options: nosniff`, `Referrer-Policy`, `Permissions-Policy`, HSTS. Заголовок `X-Powered-By` скрыт.
- **Honeypot** на форме входа + блокировка перебора пароля.
- **Секрет сессии**: в продакшене приложение не стартует без сильного `SESSION_SECRET` (≥32 симв.). Cookie сессии — `httpOnly`, `secure` по HTTPS. Шаблон переменных — в `.env.example`.
- **Страницы ошибок** без технических деталей: `app/not-found.tsx` (404), `app/error.tsx` (500).

## Структура
- `app/` — публичный каталог (`/`, `/equipment/[slug]`), вход (`/login`),
  рабочее место (`/manage/*`), правила (`/manage/admin`)
- `lib/domain.ts` — бизнес-логика (уровни доверия, баллы рейтинга, залог, расчёт стоимости)
- `lib/queries.ts`, `lib/auth.ts`, `lib/session.ts`, `lib/db.ts`, `lib/config.ts`
- `prisma/schema.prisma` — модель данных, `prisma/seed.ts` — демо-данные
- `proxy.ts` — защита разделов `/manage` и `/admin`

## Проверка бизнес-логики
```bash
npm run verify              # тесты доменных функций + цикл выдача→возврат на БД
```

## Конфигурация (`.env`)
- `NEXT_PUBLIC_WHATSAPP` — номер для приёма заявок (формат wa.me, только цифры)
- `NEXT_PUBLIC_BASE_URL` — базовый URL для QR-кодов
- `SESSION_SECRET` — секрет подписи сессий (заменить в проде)
